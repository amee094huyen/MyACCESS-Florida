# Contribution Guidelines

Thank you for your interest in contributing to the MyACCESS Florida Expense Tracker project. We welcome contributions from developers, designers, testers, and documentation writers.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Follow the project's coding standards.
4. Write clear and maintainable code.
5. Test your changes thoroughly.
6. Commit your changes with descriptive commit messages.
7. Push your branch to your forked repository.
8. Submit a Pull Request for review.

## Reporting Issues

If you discover a bug or issue:

* Check existing issues before creating a new one.
* Provide detailed information about the problem.
* Include steps to reproduce the issue.
* Mention your operating system and browser version.
* Add screenshots when applicable.

## Suggesting Features

We welcome feature suggestions that improve usability, performance, and accessibility.

When suggesting a feature:

* Clearly explain the proposed functionality.
* Describe the problem it solves.
* Include mockups or examples if available.
* Explain potential benefits for users.

## Code of Conduct

All contributors are expected to:

* Be respectful and professional.
* Welcome constructive feedback.
* Promote an inclusive environment.
* Avoid offensive or discriminatory behavior.
* Collaborate positively with community members.

## Review Process

Every Pull Request will undergo review before merging.

Review criteria include:

* Code quality and readability.
* Project consistency.
* Functionality and testing.
* Security considerations.
* Documentation updates when required.

Maintainers may request modifications before approval. Once approved, changes will be merged into the main branch.
